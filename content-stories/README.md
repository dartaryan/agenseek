## 🎨 עקרונות יצירת תוכן

### 1. כתיבה ברורה ופשוטה
- משפטים קצרים
- מושגים מוסברים
- דוגמאות מעשיות
- Hebrew בלבד (כולל קוד comments)

### 2. מבנה עקבי
- כל מדריך מתחיל ב-H1
- סקשנים עיקריים ב-H2
- תת-סקשנים ב-H3
- Table of Contents נבנה אוטומטית מההדינגים

### 3. שימוש חכם בבלוקים
- **Callout (info):** הסברים חשובים
- **Callout (warning):** אזהרות ושגיאות נפוצות
- **Callout (success):** טיפים והצלחות
- **Code:** דוגמאות קוד עם syntax highlighting
- **Accordion:** תוכן ארוך שמתקפל
- **Tabs:** חלופות שונות (למשל, לפי תפקיד)
- **List:** צעדים או נקודות עיקריות

### 4. אורך מתאים
- **Beginner:** 15-30 דקות (2000-4000 מילים)
- **Intermediate:** 30-45 דקות (4000-6000 מילים)
- **Advanced:** 45-60 דקות (6000-8000 מילים)

### 5. טאגים רלוונטיים
- 3-6 טאגים לכל מדריך
- בעברית
- מסייעים לחיפוש והמלצות

## 🔒 כללי זהב - חובה לקרוא!

### ✅ מה מותר:
1. ליצור JSON חדש בפורמט הנכון
2. להוסיף מטא-דאטה לקטלוג (index.json)
3. להשתמש בכל 14 הבלוקים הזמינים
4. לכתוב בעברית שוטפת
5. להוסיף דוגמאות קוד

### ❌ מה אסור:
1. לשנות את המבנה של הפורמט JSON
2. להוסיף בלוקים חדשים שלא קיימים
3. לשנות את הקוד של האתר
4. לשנות את הטייפים של content-blocks.ts
5. להשתמש באנגלית (מלבד קוד וטרמינולוגיה טכנית)

### ⚠️ נקודות קריטיות:
- **metadata.slug** חייב להיות זהה לשם הקובץ
- **metadata.id** חייב להיות זהה ל-slug
- **tableOfContents** נבנה אוטומטית מההדינגים (אבל צריך להכין אותו ידנית)
- **כל heading צריך id ייחודי** לטובת ToC
- **anchor בטוקן** צריך להתאים ל-id של ההדינג
