| Shortcut | Action | Context |
|----------|--------|---------|

## Tips for Keyboard Users

### 1. **Focus Indicators**
All interactive elements have a visible emerald outline when focused. If you don't see where focus is, press `Tab` again.

### 2. **Skip Repetitive Content**
Press `Tab` immediately after page load to reveal "Skip to main content" link.

### 3. **Command Palette is Your Friend**
Press `Ctrl/Cmd + K` to access all actions quickly without navigating menus.

### 4. **Use Arrow Keys in Lists**
When navigating dropdowns, search results, or command palette, use `↑`/`↓` arrow keys.

### 5. **Escape to Close**
`Escape` key closes almost any modal, dropdown, or overlay.

### 6. **Form Navigation**
Use `Tab` to move between form fields. `Enter` submits forms (when on submit button).

### 7. **First-Time Help**
The keyboard shortcuts modal appears automatically on first visit. Press `?` anytime to see it again.

## Screen Reader Users

### Recommended Screen Readers

- **Windows**: NVDA (free) or JAWS
- **macOS/iOS**: VoiceOver (built-in)
- **Android**: TalkBack (built-in)
- **Linux**: Orca

### Navigation by Landmarks

- Press landmark navigation key (usually `D` in NVDA/JAWS) to jump between:
  - Banner (header)
  - Navigation (sidebar)
  - Main content
  - Complementary (sidebar panels)
  - Content information (footer)

### Headings Structure

- Pages have logical heading hierarchy (h1 → h2 → h3)
- Press `H` to navigate by headings
- Press `1-6` to jump to specific heading levels

## Keyboard Navigation Best Practices

1. **Always show focus**: Focus indicators are visible on all interactive elements
2. **Logical tab order**: Elements are focusable in a logical sequence
3. **No keyboard traps**: You can always navigate away from any element
4. **Shortcuts don't conflict**: Custom shortcuts use `Ctrl`/`Cmd`/`Alt` modifiers
5. **Escape works everywhere**: Close modals, dropdowns, and overlays with `Escape`
