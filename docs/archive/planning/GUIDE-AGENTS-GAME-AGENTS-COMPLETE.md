## 🎯 תוכן המדריך

המדריך מכסה 6 סקשנים מרכזיים:

### 1. מבוא: אג'נטי המשחקים
- למה משחקים צריכים אג'נטים מיוחדים?
- מתי להשתמש באג'נטי המשחקים?
- סוגי המשחקים המתאימים

### 2. Game Designer - מעצב המשחק
- מי זה Game Designer ומה הוא עושה
- וורקפלואים עיקריים:
  - `*game-brief` - יצירת בריף משחק
  - `*gdd` - כתיבת Game Design Document
  - `*narrative` - בניית סיפור ונרטיב
- דוגמאות מעשיות
- טיפים לעבודה

### 3. Game Architect - ארכיטקט המשחק
- מי זה Game Architect ומה הוא עושה
- וורקפלואים עיקריים:
  - ניתוח דרישות ארכיטקטוניות
  - תכנון ארכיטקטורה
  - `*tech-spec` - כתיבת מפרט טכני
- החלטות ארכיטקטוניות נפוצות (Tabs):
  - בחירת מנוע
  - Design Patterns
  - שיקולי ביצועים
- דוגמאות מעשיות
- טיפים לעבודה

### 4. Game Dev - מפתח המשחק
- מי זה Game Dev ומה הוא עושה
- וורקפלואים עיקריים:
  - מימוש פיצ'רים
  - איתור ותיקון באגים
  - אופטימיזציה וביצועים
- מומחיות (Tabs):
  - Gameplay Code
  - Game Systems
  - Tools & Debug
- דוגמת קוד: Water Flow System (C#)
- טיפים לעבודה

### 5. תהליך מלא: מרעיון למשחק עובד
- הזרימה בין שלושת האג'נטים
- 5 שלבי פיתוח:
  - Concept (Game Designer)
  - Design (Game Designer)
  - Architecture (Game Architect)
  - Development (Game Dev)
  - Polish & Testing
- Timeline מלא למשחק קטן (טבלה)
- תקשורת בין האג'נטים

### 6. שאלות נפוצות (Accordion)
- האם אני חייב להשתמש בשלושת האג'נטים?
- האם אני צריך לדעת לתכנת?
- איזה מנוע משחקים הכי טוב?
- כמה זמן לוקח לבנות משחק?
- האם Game Designer יכול לכתוב את הסיפור?
- מה עם גרפיקה ואודיו?
- האם יש תמיכה ב-multiplayer?
- איך עובדים עם הוורקפלואים האחרים?
