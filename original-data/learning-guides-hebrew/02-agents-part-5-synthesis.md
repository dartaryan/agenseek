## שיתוף פעולה רב-אגנטי

### מה זה שיתוף פעולה רב-אגנטי?

לעתים קרובות, משימה אחת דורשת **יותר מאגנט אחד**. במקום לנסות לעשות הכל עם אגנט אחד, BMAD מאפשר לכם **לשלב מספר אגנטים בצורה חכמה** - כל אחד תורם את המומחיות שלו.

### Party Mode - צ'אט קבוצתי עם כל האגנטים 🎉

**מה זה Party Mode?**

Party Mode זו פיצ'ר שמאפשר לכם **לטעון מספר אגנטים בו-זמנית בצ'אט אחד**. כל האגנטים רואים את השיחה ויכולים לתרום את נקודת המבט שלהם.

**מתי להשתמש ב-Party Mode?**

✅ כאשר יש נושא מורכב הדורש נקודות מבט מרובות  
✅ כאשר רוצים "ישיבת צוות" עם מומחים שונים  
✅ כאשר צריך החלטה שמשפיעה על תחומים רבים  
✅ כאשר רוצים לקבל משוב מתפקידים שונים

**איך להשתמש ב-Party Mode?**

1. טען את BMad Master: `@bmad-master hi`
2. בחר: `*party-mode`
3. BMad Master יטען את כל האגנטים הרלוונטיים
4. תשאל שאלה - כל האגנטים יגיבו מנקודת מבטם

**דוגמה:**
```
אתה: "אנחנו שוקלים להוסיף פיצ'ר חדש - משתמשים יכולים לשתף תוכן ברשתות חברתיות. מה אתם חושבים?"

John (PM): "אני צריך לדעת - מה ה-Job-to-be-Done? למה משתמשים רוצים לשתף?"

Sally (UX): "איך זה ישתלב בזרימת המשתמש? איפה הכפתור?"

Winston (Architect): "יש לנו API לרשתות חברתיות? או צריך לבנות?"

Amelia (DEV): "כמה זמן זה ייקח? יש כאן OAuth integrations..."

Murat (TEA): "איך נבדוק את זה? צריך mocks לרשתות?"

Bob (SM): "זה נשמע כמו 2-3 Stories. בואו נפרק."
```

**כל אחד תורם את המבט שלו - ואתם מקבלים תמונה מלאה!**

## התאמה אישית של אגנטים

### למה להתאים אישית?

האגנטים של BMAD מגיעים עם **הגדרות ברירת מחדל מעולות**, אבל לפעמים אתם רוצים:

- 🌍 **לשנות שפה** - לעבוד בעברית/אנגלית/שפה אחרת
- 🎭 **לשנות אישיות** - פורמלי יותר או יותר casual
- 📝 **לשנות סגנון פלט** - פורמט מסמכים אחר
- 🏢 **להוסיף נהלים של החברה** - תהליכים ספציפיים

### איפה נמצאים קבצי ההתאמה?

קבצי ההתאמה נמצאים ב:
```
bmad/_cfg/agents/*.customize.yaml
```

**דוגמאות:**
- `bmad/_cfg/agents/bmm-pm.customize.yaml` - התאמת John (PM)
- `bmad/_cfg/agents/bmm-dev.customize.yaml` - התאמת Amelia (DEV)
- `bmad/_cfg/agents/cis-brainstorming-coach.customize.yaml` - התאמת Carson

### מה אפשר להתאים?

#### 1. שפת תקשורת
```yaml
communication_language: Hebrew
document_output_language: Hebrew
```

**תוצאה:** האגנט ידבר איתך בעברית ויכתוב מסמכים בעברית.

#### 2. שינוי אישיות
```yaml
communication_style_override: |
  תקשורת ישירה וקצרה. ללא מעטפות. ישר לעניין.
```

**תוצאה:** האגנט ישנה את סגנון התקשורת שלו.

#### 3. הוספת נהלים של החברה
```yaml
additional_guidelines: |
  - כל PRD חייב לעבור Code Review
  - השתמש תמיד ב-Jira format לגבי Stories
  - כל ארכיטקטורה צריכה אישור Security Team
```

**תוצאה:** האגנט יכלול את הנהלים בעבודתו.

#### 4. שינוי שם האגנט
```yaml
override_name: "משה"
```

**תוצאה:** במקום "John" האגנט יקרא "משה".

### איך לערוך קובץ התאמה?

1. **מצא את הקובץ** - `bmad/_cfg/agents/[agent-name].customize.yaml`
2. **פתח עורך טקסט** - כל עורך (VS Code, Notepad++, וכו')
3. **ערוך את השדות** - שנה את מה שצריך
4. **שמור את הקובץ**
5. **טען את האגנט מחדש** - השינויים יחולו מיד

### התאמות נפוצות

#### התאמה 1: עברית מלאה
```yaml
communication_language: Hebrew
document_output_language: Hebrew
override_name: "יוחנן"  # John
```

#### התאמה 2: פורמליות גבוהה
```yaml
communication_style_override: |
  תקשורת רשמית ומקצועית ברמה גבוהה.
  שימוש בלשון רבים מכובדת.
  הימנעות מז'רגון לא מקצועי.
```

#### התאמה 3: חברה ספציפית
```yaml
additional_guidelines: |
  - חברה: Tech Corp Ltd
  - כל מסמך חייב לכלול לוגו בראש
  - פורמט תאריכים: DD/MM/YYYY
  - שעות עבודה: 09:00-18:00
  - Sprint length: 2 weeks (not 1)
```

### ⚠️ זהירות!

- **אל תמחקו שדות חובה** - תוסיפו, אל תמחקו
- **גבו לפני שינויים** - העתיקו את הקובץ המקורי
- **בדקו שהשינויים עובדים** - טענו את האגנט ובדקו
- **YAML רגיש לרווחים** - שימו לב ל-indentation
