### 🚀 שעה 2 (10:00-11:00): workflow-init והבנת המסלולים

**מטרת השעה:** להריץ את workflow-init, ליצור את קובץ ה-status, ולהבין את 3 המסלולים השונים.

#### דקות 0-20: הרצת workflow-init

workflow-init הוא הוורקפלואו הראשון שצריך להריץ בכל פרויקט חדש. הוא מסייע לכם להגדיר את הפרויקט ולבחור מסלול עבודה.

**צעדים:**

1. ודאו ש-BMad Master פעיל בצ'אט
2. בחרו באופציה: `*list-workflows`
3. מצאו את `workflow-init`
4. הריצו אותו

**מה קורה:**
- BMad Master שואל שאלות על הפרויקט שלכם
- סוג הפרויקט (אפליקציה חדשה, תחזוקה, תכונה חדשה)
- גודל הצוות
- טכנולוגיות
- מועד יעד

**תרגיל מעשי 2:**
הריצו workflow-init עם פרויקט דמה:
- **שם פרויקט:** "Todo App"
- **סוג:** אפליקציה חדשה
- **צוות:** 1-2 מפתחים
- **טכנולוגיות:** React, Node.js
- **מועד יעד:** חודשיים

#### דקות 20-40: הבנת 3 המסלולים

BMAD-METHOD מציע 3 מסלולים לפי סוג הפרויקט:

**🎯 מסלול א': פיתוח מאפס (From Scratch)**
- מתאים ל: פרויקטים חדשים לגמרי
- מתחיל מ: רעיון ראשוני
- כולל: product-brief → tech-spec → architecture → development
- משך זמן טיפוסי: 2-6 חודשים

**🔧 מסלול ב': הוספת תכונה (Feature Addition)**
- מתאים ל: הוספת יכולות למערכת קיימת
- מתחיל מ: feature-spec
- כולל: design → implementation → testing
- משך זמן טיפוסי: 2-8 שבועות

**🛠️ מסלול ג': תחזוקה ותיקונים (Maintenance)**
- מתאים ל: באגים, שיפורים, רפקטורינג
- מתחיל מ: bug-report או improvement-request
- כולל: analysis → fix → validation
- משך זמן טיפוסי: ימים עד שבועיים

**📊 טבלת השוואה:**

| קריטריון | מסלול א' | מסלול ב' | מסלול ג' |
|---------|---------|---------|---------|
| מורכבות | גבוהה | בינונית | נמוכה-בינונית |
| אג'נטים מעורבים | 12-15 | 6-10 | 3-6 |
| וורקפלואים | 15-25 | 8-15 | 3-8 |
| משך זמן | חודשים | שבועות | ימים-שבועיים |
| תיעוד מלא | ✅ חובה | ✅ מומלץ | ⚠️ חלקי |

#### דקות 40-60: יצירת קובץ workflow-status

אחרי שהרצנו workflow-init, המערכת יוצרת קובץ `workflow-status.json` בתיקיית הפרויקט.

**מבנה הקובץ:**

```json
{
  "project_name": "Todo App",
  "track": "from-scratch",
  "current_phase": "phase-0",
  "completed_workflows": [],
  "next_workflows": [
    "product-brief",
    "stakeholder-interviews"
  ],
  "team_members": [
    {
      "name": "User",
      "role": "Full Stack Developer"
    }
  ],
  "created_at": "2025-11-05T10:30:00Z",
  "last_updated": "2025-11-05T10:30:00Z"
}
```

**למה זה חשוב:**
- 📍 מעקב אחר התקדמות הפרויקט
- 🔄 אג'נטים יכולים לראות היכן אתם
- ✅ ניהול משימות והשלמות
- 📅 תכנון הצעדים הבאים

**מה האג'נט יעשה:**

- ייצור מסמך product-brief מלא

**תרגיל מעשי 3:**

```

**קובץ:** `bmad/bmm/agents/analyst.md`

**איך משתמשים:**

1. טענו את Analyst Agent
2. העבירו לו את ה-product brief שה-PM יצר

**מה האג'נט יעשה:**

- יגדיר תהליכי עבודה (flows)

- יכתוב acceptance criteria
- ייצור מסמך functional-spec מלא

**תרגיל מעשי 4:**
```

Analyst: "אני רואה 5 תכונות עיקריות..."
[תנו לו להשלים את הניתוח]

**פלטים שתקבלו:**
- `{output_folder}/functional-spec.md`
- `{output_folder}/user-flows.md`
- `{output_folder}/acceptance-criteria.md`

#### דקות 40-60: BMad Master - המתאם

**תפקיד:** מתאם וורקפלואים, מנהל אג'נטים אחרים, מבצע משימות.

**קובץ:** `bmad/core/agents/bmad-master.md`

**למה הוא מיוחד:**

- 🔄 מריץ וורקפלואים
- 📋 מנהל רשימות משימות

## 📝 דף תרגיל ליום 1

### תרגיל מסכם: Todo App מתחילה ועד סוף

**זמן משוער:** 60-90 דקות

**משימה:** צרו todo app פשוט עם BMAD-METHOD.

**צעדים:**

1. **Phase 0: Initialization (10 דק')**
   ```
   - הריצו workflow-init
   - הגדירו פרויקט: "My Todo App"
   - בחרו מסלול: From Scratch
   ```

2. **Phase 1: Product Definition (20 דק')**
   ```
   - טענו PM Agent
   - צרו product-brief עם:
     * 3 תכונות עיקריות
     * 2 user personas
     * Success metrics
   ```

3. **Phase 2: Technical Spec (20 דק')**
   ```
   - טענו Analyst Agent
   - צרו functional-spec
   - טענו Architect Agent
   - צרו tech-spec עם:
     * Tech stack: React + Firebase
     * 3 main components
     * Data model
   ```

4. **Phase 3: Design (15 דק')**
   ```
   - טענו UX Designer Agent
   - צרו design-system
   - הגדירו 3 מסכים עיקריים
   ```

5. **Phase 4: Development Plan (10 דק')**
   ```
   - טענו Scrum Master Agent
   - צרו sprint plan
   - חלקו ל-3 sprints
   ```

6. **סיכום (5 דק')**
   ```
   - סקרו את כל המסמכים
   - ודאו שהכל עקבי
   - שמרו בתיקייה מסודרת
   ```

**בונוס:** אם יש זמן, התחילו ליישם קומפוננט אחד עם Developer Agent!

## 🆘 נתקעתם? עזרה מהירה

### בעיות התקנה

**בעיה:** "node: command not found"
```bash
# פתרון:
1. וודאו שהתקנתם Node.js
2. הפעילו מחדש את הטרמינל
3. נסו: node --version
4. אם עדיין לא עובד - התקינו מחדש Node.js
```

**בעיה:** "npm install נכשל"
```bash
# פתרון:
1. נסו: npm cache clean --force
2. מחקו node_modules: rm -rf node_modules
3. נסו שוב: npm install
4. אם עדיין לא עובד - בדקו חיבור לאינטרנט
```

### בעיות אג'נטים

**בעיה:** "האג'נט לא מגיב"
```
פתרון:
1. ודאו שהעתקתם את כל תוכן ה-YAML/MD
2. נסו שוב עם הודעה פשוטה: "hi"
3. בדקו שקובץ config.yaml קיים
4. הפעילו מחדש את ה-IDE
```

**בעיה:** "האג'נט מייצר פלט באנגלית"
```
פתרון:
1. פתחו config.yaml
2. ודאו: communication_language: "Hebrew"
3. שמרו את הקובץ
4. טענו מחדש את האג'נט
```

### בעיות וורקפלואים

**בעיה:** "workflow-init לא מתחיל"
```
פתרון:
1. ודאו ש-BMad Master טעון
2. נסו: "*list-workflows"
3. אם הרשימה ריקה - בדקו את הנתיב לתיקייה
4. ודאו שאתם בתיקיית הפרויקט הנכונה
```

**בעיה:** "הפלטים לא נשמרים"
```
פתרון:
1. בדקו את output_folder ב-config.yaml
2. ודאו שיש הרשאות כתיבה
3. נסו ליצור את התיקייה ידנית
4. בדקו שאין שגיאות בלוג
```
