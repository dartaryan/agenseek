## 🎯 מבוא לשבוע הראשון

### למה שבוע ראשון?

השבוע הראשון שלכם עם BMAD-METHOD הוא **קריטי להצלחה לטווח ארוך**. במהלך שבוע זה תעברו מ"מה זה BMAD?" ל"אני משתמש בBMAD כל יום!"

### מטרות השבוע

בסוף השבוע הראשון, תוכלו:

✅ להתקין ולהגדיר את BMAD במחשב שלכם  
✅ להבין את מבנה המערכת והפילוסופיה שמאחוריה  
✅ לעבוד עם 5-7 אג'נטים בביטחון  
✅ להריץ 5-10 וורקפלואים שונים  
✅ להשלים פרויקט קטן מקצה לקצה  
✅ להתאים את BMAD לצרכים שלכם  
✅ לדעת איך לפתור בעיות נפוצות  

### השקעת זמן

**זמן מומלץ:** 8-12 שעות במהלך השבוע

- יום 1: 3-4 שעות (ההתקנה והיסודות)
- יום 2: 2-3 שעות (אג'נטים)
- יום 3: 2-3 שעות (וורקפלואים)
- יום 4: 2-3 שעות (תרגול)
- יום 5: 1-2 שעות (סיכום ותכנון)

**אין צורך לעשות הכל ביום אחד!** פזרו את הלמידה לאורך השבוע.

### מבנה הלמידה

כל יום כולל:

📖 **תיאורית** - מה תלמדו  
👨‍💻 **מעשית** - תרגילים אינטראקטיביים  
✅ **אימות** - איך תדעו שהצלחתם  
🎯 **משימות** - מה לעשות בבית

### 🎭 Party Mode - צ'אט קבוצתי

**מה זה Party Mode?**

זה מצב מיוחד שבו כמה אג'נטים נמצאים בצ'אט **במקביל** - כמו ישיבת צוות!

**איך להפעיל:**

```
1. טענו את BMad Master
2. שלחו: *party-mode
3. BMad Master יטען את כל האג'נטים הרלוונטיים
4. עכשיו אתם יכולים לדבר עם כולם!
```

**דוגמה לשימוש:**

```
אתם: "בואו נתכנן ביחד את אפליקציית התזכורות"

John (PM): "אני אתחיל - מה קהל היעד שלנו?"

Mary (Analyst): "אני אנתח את הדרישות אחרי ש-John יסיים"

Winston (Architect): "אני כבר חושב על הארכיטקטורה..."

Lisa (UX): "תנו לי לעצב את החוויה!"
```

**מתי להשתמש ב-Party Mode:**

✅ כשצריכים החלטות מהירות מכמה פרספקטיבות  
✅ בתכנון ראשוני של פרויקט  
✅ כשיש בעיה מורכבת שצריכה מספר מומחיות  
✅ ב-brainstorming sessions  

**מתי לא להשתמש:**

❌ כשצריכים פלט מסודר (כמו מסמך PRD מלא)  
❌ כשיש יותר מדי אג'נטים - זה נהיה מבלבל  
❌ למשימות פשוטות שאג'נט אחד יכול לטפל בהן

### 📚 מה זה Workflow?

**הגדרה פשוטה:**

Workflow = **תהליך עבודה מובנה** שמורכב ממספר שלבים, בו אג'נט אחד או יותר עוזרים לכם להשלים משימה.

**דוגמה מהחיים האמיתיים:**

כשאתם הולכים לרופא:
1. קביעת תור ← workflow step 1
2. מילוי טופס רפואי ← step 2
3. בדיקה ← step 3
4. קבלת מרשם ← step 4

ב-BMAD זה אותו עקרון - רק שה"רופא" הוא אג'נט!

**מבנה Workflow טיפוסי:**

```yaml
workflow:
  name: product-brief
  description: Create a product brief document
  
  steps:
    1. Load PM Agent
    2. Ask user about product
    3. Generate product brief
    4. Save to docs/prd/product-brief.md
```

### 🔥 Workflows קריטיים שכולם צריכים

#### 1. workflow-init - תמיד ראשון!

**מה הוא עושה:**

- יוצר את קובץ workflow-status.yaml
- ממליץ על מסלול עבודה (track)
- מתכנן את הצעדים הבאים

**מתי להריץ:**
- **תמיד** בתחילת פרויקט חדש
- לפני שאתם מריצים workflows אחרים

**איך להריץ:**
```
1. טענו BMad Master
2. שלחו: *workflow-init
או
2. שלחו: run workflow-init
```

**תרגיל:**
```
הריצו workflow-init עם פרויקט דמה:
- שם: "ToDo App Pro"
```

## ❓ שאלות נפוצות - שבוע ראשון

### בעיות טכניות

**ש: האג'נט לא מגיב כשאני שולח הודעה**

ת: בדקו:
1. שהעתקתם את כל תוכן קובץ האג'נט
2. שקובץ config.yaml קיים
3. נסו לטעון מחדש את האג'נט
4. בדקו שה-IDE שלכם תומך באג'נטים

**ש: workflow-init לא עובד**

ת: וודאו:
1. ש-BMad Master טעון
2. שאתם בתיקיית הפרויקט הנכונה
3. שיש לכם הרשאות כתיבה
4. נסו: `*list-workflows` לראות שהוא זמין

**ש: המסמכים לא נשמרים**

ת: בדקו:
1. את ה-`output_folder` ב-config.yaml
2. שיש הרשאות כתיבה לתיקייה
3. שהתיקייה קיימת (או שBMAD יכול ליצור אותה)
4. שאין שגיאות בלוג

### שאלות שימוש

**ש: איזה אג'נט להשתמש מתי?**

ת: כלל אצבע:
- **תכנון** → PM, Analyst
- **טכני** → Architect, Developer
- **עיצוב** → UX Designer
- **תיעוד** → Tech Writer
- **ניהול** → Scrum Master, BMad Master

**ש: כמה זמן לוקח workflow?**

ת: תלוי ב-workflow:
- פשוט (dev-story): 5-10 דקות
- בינוני (product-brief): 15-20 דקות
- מורכב (prd, architecture): 30-60 דקות

**ש: אני צריך להריץ את כל ה-workflows?**

ת: **לא!** תלוי בפרויקט:
- פרויקט קטן: 3-5 workflows
- פרויקט בינוני: 8-12 workflows
- פרויקט גדול: 15-25 workflows

**ש: מה עושים כש-workflow נתקע?**

ת:
1. בדקו את workflow-status
2. ודאו שיש את כל הקבצים הנדרשים
3. נסו להריץ מחדש
4. בקשו עזרה מBMad Master

### שאלות למידה

**ש: לא הצלחתי לסיים את כל השבוע**

ת: **זה בסדר גמור!** BMAD זה מערכת עשירה. קחו את הזמן שאתם צריכים. אפשר לפזר את הלמידה על שבועיים.

**ש: אני מרגיש/ה המום/ה מכל האפשרויות**

ת: זה נורמלי! התמקדו ב:
1. 3 אג'נטים בלבד
2. 3 workflows בלבד
3. תרגול על אותם דברים כמה פעמים
4. אחר כך הרחבו

**ש: מתי אתחיל לעבוד על פרויקטים אמיתיים?**

ת: **כבר בסוף השבוע הראשון!** אחרי יום 4, אתם אמורים להיות מוכנים לעבוד על פרויקטים קטנים אמיתיים.
