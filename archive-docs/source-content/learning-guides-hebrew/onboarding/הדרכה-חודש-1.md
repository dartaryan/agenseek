## 🎯 סקירה כללית

### מטרות החודש הראשון

בסיום החודש הראשון שלכם עם BMAD-METHOD, תהיו:

✅ **מיומנים** בעבודה עם 10-15 אג'נטים שונים  
✅ **בקיאים** ב-15+ workflows שונים  
✅ **מסוגלים** להשלים פרויקטים מקצה לקצה  
✅ **בטוחים** בהתאמה אישית של BMAD לצרכים שלכם  
✅ **מוכנים** ללמד אחרים ולשתף ידע  
✅ **משתמשים** ב-BMAD באופן יומיומי בעבודה אמיתית  

### ציר הזמן והשקעה

**השקעת זמן מומלצת:** 10-15 שעות בשבוע

| שבוע | התמקדות | זמן | רמה |
|------|---------|-----|-----|
| **1** | יסודות והתקנה | 12 שעות | מתחיל |
| **2** | התמחות בתפקיד | 10 שעות | בסיסי |
| **3** | שימוש מתקדם | 12 שעות | מתקדם |
| **4** | מומחיות ושיתוף | 8 שעות | מומחה |

**סה"כ:** 40-45 שעות במהלך החודש

### מסלול ההתקדמות

```
מתחיל → יסודות → בסיסי → מתקדם → מומחה
  ↓         ↓        ↓         ↓         ↓
יום 1    שבוע 1   שבוע 2   שבוע 3   שבוע 4
```

**רמות מיומנות:**

1. **מתחיל (יום 1-3):** מכיר את המושגים, מתקין את המערכת
2. **יסודות (שבוע 1):** עובד עם 5 אג'נטים, 5 workflows
3. **בסיסי (שבוע 2):** מתמחה בתפקיד, workflows ספציפיים
4. **מתקדם (שבוע 3):** workflows מורכבים, התאמות אישיות
5. **מומחה (שבוע 4):** שליטה מלאה, מלמד אחרים

## 📅 שבוע 3: שימוש מתקדם

### 🎯 מטרות השבוע

השבוע השלישי הוא על **מיומנות מתקדמת** - לעבור מ"שימוש בסיסי" ל"שימוש מקצועי".

**מה תשיגו השבוע:**

✅ שימוש ב-multi-agent workflows (כמה אג'נטים ביחד)  
✅ התאמה אישית מתקדמת של BMAD  
✅ אינטגרציה עם כלי עבודה קיימים  
✅ עבודה בצוות עם BMAD  
✅ שליטה בworkflows מורכבים  
✅ יצירת custom workflows (אופציונלי)  

### 🔄 Multi-Agent Workflows

**מה זה?**

במקום לעבוד עם אג'נט אחד בכל פעם, משתמשים ב-**כמה אג'נטים ביחד** בתהליך אחד.

#### תרחיש 1: תכנון פיצ'ר מורכב (2-3 שעות)

**זרימת עבודה:**

```
1. Chat עם PM Agent:
   - תיאור הפיצ'ר
   - יצירת product brief קצר
   ↓
2. Chat עם Analyst Agent:
   - קריאת product brief
   - ניתוח מעמיק
   - זיהוי edge cases
   ↓
3. Chat עם Architect Agent:
   - קריאת הניתוח
   - תכנון ארכיטקטורה
   - זיהוי dependencies
   ↓
4. Chat עם UX Designer:
   - קריאת כל התיעוד
   - עיצוב UI/UX
   ↓
5. Chat עם SM Agent:
   - קריאת הכל
   - פירוק ל-Stories
   ↓
6. Chat עם DEV Agent:
   - מימוש Story ראשון
```

**למה זה שונה משבוע 1-2?**
- עכשיו אתם **מתזמרים** בין אג'נטים
- כל אג'נט מקבל את התוצרים של הקודם
- תהליך מלא מקצה לקצה

#### תרחיש 2: Brownfield - הוספת feature לקוד קיים (2-3 שעות)

**זרימת עבודה:**

```
1. document-project:
   - הבנת הקוד הקיים
   - זיהוי patterns
   ↓
2. Chat עם Architect + קריאת docs/index.md:
   - איפה ואיך להוסיף feature
   - תכנון integration
   ↓
3. Chat עם DEV + כל הקונטקסט:
   - implementation
   - עקבי עם הקוד הקיים
   ↓
4. Chat עם TEA:
   - test strategy
   - regression tests
```

#### תרחיש 3: Code Quality Improvement (1-2 שעות)

**זרימת עבודה:**

```
1. Chat עם DEV - code-review:
   - סריקת הקוד
   - זיהוי בעיות
   ↓
2. Chat עם Architect:
   - קריאת code review
   - המלצות ארכיטקטוניות
   ↓
3. Chat עם DEV - refactor:
   - תכנית refactoring
   - מימוש
   ↓
4. Chat עם TEA:
   - ודא שלא שברנו דברים
```

#### Git Integration

**Best Practices:**

```bash
# Branch naming convention
git checkout -b feature/story-{number}-{short-desc}

# Commit messages

- Implemented task 1
- Implemented task 2

Generated with BMAD DEV agent"

# Before merge

git pull
git checkout feature/story-X

# Resolve conflicts with DEV agent if needed
git push
```

#### CI/CD Integration

**דוגמה: GitHub Actions**

```yaml
# .github/workflows/bmad-quality.yml
name: BMAD Quality Check

on: [push, pull_request]

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Install dependencies
        run: npm install
      
      - name: Lint
        run: npm run lint
      
      - name: Test
        run: npm test
      
      - name: Build
        run: npm run build
```
