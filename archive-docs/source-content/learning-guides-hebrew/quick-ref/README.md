# כרטיסי עזר מהירים (Quick Reference)

## 📋 מה יש בתיקייה הזו?

כרטיסי עזר מהירים שמרכזים מידע חשוב בפורמט קריא וזמין:

### ⚡ סוגי כרטיסי עזר:

1. **כרטיסי אג'נטים** - עמוד אחד לכל אג'נט
2. **כרטיסי וורקפלואים** - רשימה מהירה של כל וורקפלואו
3. **טבלאות השוואה** - מה להשתמש ומתי
4. **פקודות נפוצות** - כל הפקודות במקום אחד
5. **פתרון בעיות** - מדריך SOS מהיר

### 5. כרטיס פתרון בעיות SOS

```
╔══════════════════════════════════════════════════════════╗
║              🆘 פתרון בעיות מהיר                        ║
╚══════════════════════════════════════════════════════════╝

❌ בעיה: האג'נט לא עובד
   ✅ פתרון:
      1. פתחו chat חדש (Fresh Chat)
      2. טענו את קובץ האג'נט מחדש
      3. וודאו שה-IDE מזהה אותו

❌ בעיה: "לא יודע מה לעשות"
   ✅ פתרון:
      הריצו: *workflow-status
      המערכת תגיד מה הצעד הבא

❌ בעיה: הוורקפלואו תקוע
   ✅ פתרון:
      1. סגרו את הצ'אט
      2. פתחו chat חדש
      3. התחילו את הוורקפלואו מחדש

❌ בעיה: AI מזה (Hallucination)
   ✅ פתרון:
      תמיד השתמשו ב-Fresh Chat
      כל וורקפלואו = צ'אט חדש!

❌ בעיה: התקנה נכשלה
   ✅ פתרון:
      1. וודאו Node.js v20+
      2. הריצו: npm cache clean --force
      3. נסו שוב: npx bmad-method@alpha install

❌ בעיה: לא מוצא קבצים
   ✅ פתרון:
      וודאו שאתם בתיקיית הפרויקט:
      cd /path/to/your/project
```

### 6. כרטיס בחירת מסלול - Track Selector

```
╔══════════════════════════════════════════════════════════╗
║              איזה מסלול (Track) לבחור?                 ║
╚══════════════════════════════════════════════════════════╝

⚡ Quick Flow (Level 0-1):
   ✅ באג בודד
   ✅ תכונה קטנה (1-5 Stories)
   ✅ שינוי מוגדר היטב
   📝 רק Tech-Spec
   ⏱️ שעות עד יום אחד

🚀 BMad Method (Level 2-3):
   ✅ מוצר חדש
   ✅ פלטפורמה
   ✅ תכונות מרובות (10-50 Stories)
   📝 PRD + Architecture
   ⏱️ 1-3 ימי תכנון

🏢 Enterprise Method (Level 4):
   ✅ מערכת ארגונית
   ✅ Multi-tenant
   ✅ דרישות אבטחה/רגולציה
   📝 PRD + Arch + Security/DevOps
   ⏱️ 3-7 ימי תכנון

🤔 לא בטוחים?
   הריצו workflow-init - המערכת תמליץ!
```

   □ workflow-init        אתחול פרויקט

   bmad/bmm/agents/pm.md

   ✓ הגדרת דרישות
   ✓ פירוק Epics

   • תמיד התחילו עם workflow-init
   • השתמשו ב-Fresh Chat לכל וורקפלואו
   • תכינו מידע על הפרויקט מראש

🤝 עובד טוב עם:
   → Analyst (מחקר ראשוני)
   → Architect (לאחר ה-PRD)
   → SM (פירוק Stories)

## 🖨️ איך להשתמש בכרטיסים?

### לעבודה יומיומית:
1. **הדפיסו** את הכרטיסים החשובים
2. **תלו** ליד העמדה
3. **השתמשו** כאשר צריכים תזכורת מהירה

### ללמידה:
1. **קראו** כל בוקר כרטיס אחד
2. **תרגלו** את מה שכתוב
3. **שנינו** את הפקודות החשובות

### לצוות:
1. **שתפו** עם חברים לעבודה
2. **הדפיסו** לכל אחד
3. **צרו** לוח מרכזי בחדר הצוות
