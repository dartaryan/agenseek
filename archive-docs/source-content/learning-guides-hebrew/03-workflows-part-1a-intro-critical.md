## מבוא: מהם וורקפלואים ב-BMAD

### מה זה וורקפלואו?

**וורקפלואו (Workflow)** ב-BMAD הוא **תהליך עבודה מובנה** שמדריך אותך צעד אחר צעד דרך משימה מורכבת. כל וורקפלואו הוא כמו **מתכון מפורט** - האגנט שואל שאלות, מקבל תשובות, ועובד איתך באופן אינטראקטיבי כדי להשיג תוצאה ספציפית.

בניגוד ל**אגנטים** שהם "מי" שמבצע את העבודה, **וורקפלואים** הם "איך" העבודה מבוצעת - התהליך עצמו.

### למה צריך וורקפלואים?

**ללא וורקפלואו:**
```
אתה: "צור לי PRD למוצר החדש"
AI: "בטח! [מייצר PRD גנרי ללא מבנה]"
תוצאה: מסמך לא שלם, חסרים פרטים קריטיים ❌
```

**עם וורקפלואו:**
```
אתה: *prd
AI: "בואו נצור PRD מקיף. ספר לי על המוצר..."
    → 20 שאלות ממוקדות
    → הנחיה דרך כל סעיף
    → תבנית מוכחה
תוצאה: PRD מקצועי ושלם ✅
```

### איך וורקפלואו שונה ממשימה רגילה?

| היבט | משימה רגילה | וורקפלואו BMAD |
|------|-------------|----------------|
| **מבנה** | פתוח - AI מנחש מה אתה רוצה | מובנה - תהליך מוגדר בצעדים |
| **אינטראקציה** | שאלה אחת, תשובה אחת | דו-שיח מתמשך - AI שואל, אתה עונה |
| **איכות** | משתנה - תלוי בנוסח השאלה | עקבית - תמיד אותו תהליך מוכח |
| **תוצר** | טקסט חופשי | מסמך מובנה לפי תבנית |
| **למידה** | צריך לדעת מה לבקש | הווה מדריך ולימד אותך |

### איך מריצים וורקפלואו?

כל וורקפלואו מופעל דרך **אגנט רלוונטי** באמצעות **פקודה ייעודית**:

**דרך 1: בחירה מהתפריט**
```
אתה: @pm hi
John (PM): [מציג תפריט]
  1. Create PRD (*create-prd)
  2. Validate PRD (*validate-prd)
  ...
אתה: 1
```

**דרך 2: הפעלה ישירה**
```
אתה: @pm *create-prd
John (PM): [מתחיל את וורקפלואו PRD מיד]
```

**דרך 3: נתיב מלא** (אם לא במסך התפריט)
```
אתה: @pm /bmad:bmm:workflows:prd
```

### מחזור החיים של וורקפלואו

```
1. 🎯 טעינת אגנט
   ↓
2. 🚀 הפעלת וורקפלואו
   ↓
3. 💬 אינטראקציה (שאלות ותשובות)
   ↓
4. 📝 יצירת פלט (מסמך/קובץ)
   ↓
5. ✅ עדכון סטטוס (workflow-status)
   ↓
6. 🔄 וורקפלואו הבא
```

### סוגי וורקפלואים

**וורקפלואים קריטיים (חובה):**
- 🏗️ **workflow-init** - אתחול פרויקט
- 📊 **workflow-status** - בדיקת התקדמות
- 📋 **prd / tech-spec** - תכנון מוצר
- 🏃 **sprint-planning** - תכנון ספרינט
- 💻 **dev-story** - פיתוח Story

**וורקפלואים אופציונליים (מומלצים):**
- 🧠 **brainstorm** - יצירת רעיונות
- 🔍 **research** - מחקר
- 🎨 **create-ux-design** - עיצוב UX
- 🏗️ **architecture** - ארכיטקטורה
- 🧪 **testarch-*** - בדיקות

### כללי אצבע לעבודה עם וורקפלואים

1. **וורקפלואו אחד = צ'אט אחד** - כל וורקפלואו במסך נקי
2. **עקבו אחר הסדר** - הווקרפלואים מסודרים בסדר לוגי
3. **ענו בכנות** - ה-AI לא שופט, הוא עוזר
4. **שמרו את הפלט** - כל וורקפלואו מייצר קבצים חשובים
5. **בדקו workflow-status** - תמיד דעו איפה אתם

## workflow-init - אתחול פרויקט 🏗️

### מה זה workflow-init?

**workflow-init** הוא הוורקפלואו **הראשון והקריטי ביותר** בכל פרויקט BMAD. הוא **יוצר את נתיב העבודה המותאם אישית** עבור הפרויקט שלכם.

חשבו עליו כעל **GPS למסע הפיתוח** - הוא מבין לאן אתם הולכים ובונה את המפה המדויקת להגיע לשם.

### 🎯 מטרה

workflow-init עושה 3 דברים קריטיים:

1. **מבין את הפרויקט שלכם** - סוג, גודל, מצב (חדש/קיים)
2. **בוחר את המסלול הנכון** - Quick Flow / Method / Enterprise
3. **יוצר קובץ מעקב** - `bmm-workflow-status.yaml` שעוקב אחרי ההתקדמות

### 📥 תשומות נדרשות

- 📛 **שם הפרויקט** - איך קוראים לפרויקט
- 📖 **תיאור** - מה אתם בונים
- 🆕 **מצב** - פרויקט חדש (greenfield) או קיים (brownfield)
- 🎮 **סוג** - משחק או תוכנה רגילה

### 📤 פלט צפוי

**קובץ:** `docs/bmm-workflow-status.yaml` (או בתיקיה שהגדרתם)

**תוכן הקובץ:**
```yaml
# BMad Method Workflow Status
# Generated: 2025-11-04

project: שם הפרויקט
project_type: software  # או game
project_level: 2        # רמת מורכבות (0-4)
field_type: greenfield  # או brownfield
workflow_path: method-greenfield.yaml

workflow_status:
  # Phase 0: Discovery (Optional)
  brainstorm: optional
  research: optional
  
  # Phase 1: Analysis
  product-brief: optional
  
  # Phase 2: Planning
  prd: required
  create-ux-design: recommended
  create-epics-and-stories: required
  
  # Phase 3: Solutioning
  architecture: required
  solutioning-gate-check: required
  
  # Phase 4: Implementation
  sprint-planning: required
  # ... ועוד
```

### ⏱️ זמן משוער

**10-20 דקות** (תלוי במורכבות הפרויקט)

## workflow-status - מעקב והתקדמות 📊

### מה זה workflow-status?

**workflow-status** הוא הוורקפלואו שעונה על השאלה: **"מה אני צריך לעשות עכשיו?"**

הוא **קורא את קובץ המעקב** שנוצר ב-`workflow-init` ו**מראה לך**:
- ✅ מה כבר הושלם
- 🔄 מה הוורקפלואו הבא
- 📋 מה עוד צריך לעשות

חשבו עליו כעל **לוח מחוונים (Dashboard)** של הפרויקט.

### 🎯 מטרה

1. **הצגת סטטוס** - איפה אתם בפרויקט
2. **הכוונה** - מה הצעד הבא
3. **מניעת בלבול** - לא תשכחו איפה עצרתם

### 📥 תשומות נדרשות

**אין!** - הוורקפלואו קורא את `bmm-workflow-status.yaml` אוטומטית.

### 📤 פלט צפוי

**תצוגה אינטראקטיבית** עם:
- 📊 סטטוס נוכחי של כל הוורקפלואים
- 🎯 הוורקפלואו הבא לביצוע
- 🤖 איזה אגנט צריך לטעון
- 📝 פקודה מדויקת להפעלה

### ⏱️ זמן משוער

**30 שניות - 2 דקות** (קריאה והבנה)

### ⚠️ טעויות נפוצות

1. **לא להריץ לעתים קרובות מספיק**
   - ❌ עובדים שבועיים בלי לבדוק סטטוס
   - ✅ הריצו workflow-status **לפני כל סשן עבודה**

2. **לשכוח לעדכן ידנית**
   - ❌ עשיתם וורקפלואו אבל לא עדכן את הסטטוס
   - ✅ רוב הוורקפלואים מעדכנים אוטומטית, אבל אם עשיתם משהו ידנית - עדכנו!

3. **לא לשמור את קובץ המעקב ב-Git**
   - ❌ הקובץ `bmm-workflow-status.yaml` לא ב-Git
   - ✅ תמיד commit את הקובץ הזה - זה חלק מהפרויקט!

4. **להריץ במקום בלתי צפוי**
   - ❌ "למה workflow-status לא עובד?"
   - ✅ וודא שאתה בתיקיית הפרויקט הנכונה

### 🎓 טיפים מתקדמים

1. **הרץ בכל התחלת סשן** - תמיד הריצו בתחילת יום עבודה

2. **השתמש כ-TODO list** - זה בעצם ה-TODO list שלך לפרויקט

3. **שתף עם הצוות** - `bmm-workflow-status.yaml` מספר לכל הצוות איפה אתם

4. **עדכון ידני אם צריך** - אפשר לערוך את הקובץ ידנית (זהירות!)

5. **גיבוי** - שמרו ב-Git, עשו commit אחרי כל וורקפלואו

### 🔗 וורקפלואים קשורים

**לפני:** workflow-init (יוצר את הקובץ)  
**אחרי:** כל וורקפלואו שworkflow-status אומר לכם להריץ


**בעיה 2: "הסטטוס לא מעודכן"**
- ✅ **פתרון:** הוורקפלואו שהרצתם כנראה לא עדכן אוטומטית. עדכנו ידנית:
  ```
  workflow-status → בחרו "4. עדכן סטטוס"
  ```

**בעיה 3: "רוצה לדלג על וורקפלואו"**
- ✅ **פתרון:** ערכו ידנית את `bmm-workflow-status.yaml`:
  ```yaml
  create-ux-design: skipped  # במקום recommended
  ```

**בעיה 4: "הסטטוס מראה וורקפלואו שכבר סיימתי"**
- ✅ **פתרון:** עדכנו ידנית:
  ```yaml
  prd: docs/_prd/index.md  # הוסיפו את נתיב הקובץ
  ```

**בעיה 5: "איך לראות את כל הנתיב?"**
- ✅ **פתרון:** בterminal:
  ```bash
  cat docs/bmm-workflow-status.yaml
  ```
  או ב-workflow-status בחרו "3. הצג קובץ מלא"
