## 🎯 מבוא למסלולי הלמידה

### למה צריך מסלול למידה מותאם?

BMAD-METHOD הוא מערכת עשירה עם 19 אג'נטים ו-50+ וורקפלואים. **אתם לא צריכים ללמוד הכל!**

כל תפקיד משתמש במערכת בצורה שונה:

- 💻 **מפתח** מתמקד באג'נטים הטכניים ובוורקפלואים של פיתוח
- 📊 **מנהל מוצר** משתמש בעיקר ב-PM Agent ובוורקפלואים של תכנון
- 🎨 **מעצב UX** מתמקד ב-UX Designer Agent ובוורקפלואי create-ux-design
- 🧪 **בודק תוכנה** עובד עם TEA Agent ועם testarch workflows

**המסלול הנכון חוסך לכם שעות של למידה מיותרת!**

---

### איך להשתמש במדריך זה?

1. **מצאו את התפקיד שלכם** במטריצה למטה
2. **בדקו את רמת הניסיון שלכם** (מתחיל/בינוני/מתקדם)
3. **בחרו מסלול לפי הזמן הפנוי** (שעה/יום/שבוע/חודש)
4. **עקבו אחרי המסלול המפורט** לתפקיד שלכם
5. **בצעו את ההערכה העצמית** כדי לוודא התקדמות

- אתם מתחילים לראות את הערך האמיתי של BMAD
- אתם יכולים להסביר למישהו אחר את הבסיס

**איך לעבור לרמה הבאה:**
- הרחיבו לאג'נטים נוספים מעבר לבסיס

- שלבו כמה workflows ביחד

**זמן ממוצע ברמה:** 1-2 שבועות

- אתם יכולים לעזור לאחרים
- אתם מתחילים לראות patterns ואופטימיזציות

**איך לעבור לרמה הבאה:**

- קראו את הקוד של workflows (אם אתם טכניים)
- תרמו לקהילה - שתפו טיפים וטריקים

**זמן ממוצע ברמה:** 2-4 שבועות

**איך להמשיך לגדול:**
- צרו workflows מותאמים לארגון שלכם
- הובילו workshops ו-training sessions
- תרמו לקוד הפתוח של BMAD
